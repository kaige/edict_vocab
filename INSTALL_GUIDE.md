# 跨设备同步安装指南

## 问题说明

在开发者模式下，每台电脑加载的扩展会有不同的扩展 ID，导致无法共享云端同步数据。

## 解决方案

### 方案一：打包扩展（推荐）

#### 步骤 1: 在 PC A 上打包扩展

1. 打开 `chrome://extensions/`
2. 确保已启用"开发者模式"
3. 点击"打包扩展程序"按钮
4. 在"扩展程序根目录"中选择: `C:\code\iciba-vocab`
5. "私有密钥文件"留空（首次打包）
6. 点击"打包扩展程序"

这会生成两个文件：
- `iciba-vocab.crx` - 扩展安装包
- `iciba-vocab.pem` - 私钥文件（**非常重要，不要丢失！**）

#### 步骤 2: 复制文件到 PC B

将以下文件复制到 PC B：
- `iciba-vocab.crx`
- `iciba-vocab.pem`（如果需要在 PC B 上继续开发）

#### 步骤 3: 在两台 PC 上安装

**方法 A: 直接安装 .crx 文件**
1. 打开 `chrome://extensions/`
2. 将 `iciba-vocab.crx` 文件拖入浏览器窗口
3. 点击"添加扩展程序"

**方法 B: 继续使用开发者模式（推荐用于开发）**
1. 在两台 PC 上都删除旧的扩展
2. 确保两台 PC 的代码文件夹中都有 `iciba-vocab.pem` 文件
3. 在 PC A 上重新打包（这次选择 .pem 文件）
4. 在两台 PC 上都加载解压的扩展程序

### 方案二：使用 Chrome Web Store（最佳长期方案）

1. 注册 Chrome Web Store 开发者账号（需支付 $5 注册费）
2. 上传扩展到 Chrome Web Store
3. 可以选择"不公开"或"仅限自己使用"
4. 在两台 PC 上从 Web Store 安装

这样可以确保：
- 扩展 ID 完全一致
- 自动更新
- 无需手动管理 .pem 文件

### 方案三：共享源代码文件夹

如果两台 PC 在同一网络：
1. 将 `C:\code\iciba-vocab` 文件夹共享（使用网络共享或云同步）
2. 两台 PC 都从同一个文件夹加载扩展
3. 需要确保 .pem 文件也在共享文件夹中

## 验证安装成功

安装完成后，在两台 PC 上：

1. 打开 `chrome://extensions/`
2. 找到"iciba生词本"扩展
3. 查看"ID"字段

**两台 PC 的扩展 ID 必须完全相同！**

例如：`abcdefghijklmnopqrstuvwxyz123456`

## 测试同步

1. **在 PC A 上：**
   - 访问 iciba.com
   - 搜索一个单词（如 "hello"）
   - 打开扩展弹窗，确认单词已保存
   - 长按同步按钮，查看"云端存储: 1 个单词"

2. **等待 1-2 分钟**（Chrome 同步需要时间）

3. **在 PC B 上：**
   - 打开扩展弹窗
   - 应该能看到从 PC A 同步过来的单词
   - 长按同步按钮，确认"云端存储: 1 个单词"

## 常见问题

### Q: 打包后无法安装 .crx 文件
A: Chrome 可能阻止安装非 Web Store 的扩展。解决方法：
- 使用开发者模式加载解压的扩展
- 或上传到 Chrome Web Store

### Q: 丢失了 .pem 文件怎么办？
A: .pem 文件丢失后无法恢复相同的扩展 ID。需要：
- 重新打包生成新的 ID
- 在所有 PC 上重新安装
- 之前的云端数据会丢失（因为 ID 变了）

### Q: 同步还是不工作
A: 检查：
1. 两台 PC 的扩展 ID 是否完全相同
2. Chrome 账号是否已登录且启用同步
3. 访问 `chrome://sync-internals/` 查看同步状态
4. 查看控制台是否有错误信息

## 开发建议

如果需要在两台 PC 上同时开发：

1. 使用 Git 同步代码
2. 将 `iciba-vocab.pem` 添加到 `.gitignore`（安全考虑）
3. 手动复制 .pem 文件到两台 PC
4. 或者将 .pem 文件加密后提交到私有仓库
