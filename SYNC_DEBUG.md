# 同步问题调试指南

## 测试步骤

### 在 PC A 上：

1. **重新加载扩展**
   - 打开 `chrome://extensions/`
   - 找到 "iciba生词本" 扩展
   - 点击刷新按钮 🔄

2. **打开开发者工具**
   - 访问 https://www.iciba.com
   - 按 F12 打开开发者工具
   - 切换到 Console 标签

3. **搜索并保存一个单词**
   - 在 iciba.com 搜索框输入一个单词（例如 "test"）
   - 按回车键搜索

4. **检查控制台输出**
   查找以下关键信息：
   ```
   iciba生词本: 正在保存单词 test
   iciba生词本: 准备保存前，云端 X 个，本地 Y 个
   iciba生词本: 当前已有 N 个单词
   iciba生词本: 已保存到本地 test (共 N 个单词)
   iciba生词本: 已同步到云端 test (共 N 个单词)
   iciba生词本: 验证云端数据，实际存储了 N 个单词
   ```

   **如果看到错误：**
   ```
   iciba生词本: 同步云端失败 [错误信息]
   iciba生词本: 当前云端存储使用: XXX 字节
   ```

5. **打开扩展弹窗**
   - 点击浏览器工具栏的扩展图标
   - 右键点击弹窗 → 检查（打开弹窗的开发者工具）
   - 查看 Console 输出：
   ```
   iciba生词本: 开始加载单词...
   iciba生词本: 云端 X 个单词, 本地 Y 个单词
   iciba生词本: 云端前3个单词: [...]
   iciba生词本: 本地前3个单词: [...]
   iciba生词本: 合并后共 N 个单词
   ```

### 在 PC B 上：

1. **重新加载扩展**（同 PC A 步骤 1）

2. **等待 Chrome 同步**
   - 打开 `chrome://sync-internals/`
   - 查看 "Sync Node Browser" 标签
   - 搜索 "EXTENSION_SETTING" 查看扩展数据是否已同步

3. **打开扩展弹窗并检查**
   - 点击扩展图标
   - 右键 → 检查（打开开发者工具）
   - 查看 Console 输出，特别注意：
   ```
   iciba生词本: 云端 X 个单词, 本地 Y 个单词
   ```

4. **使用调试功能**
   - 在扩展弹窗中，长按（1秒）同步按钮
   - 会弹出详细的调试信息对话框
   - 记录显示的内容

## 常见问题诊断

### 问题 1: 云端存储为 0
**症状：** PC B 显示 "云端 0 个单词"

**可能原因：**
- Chrome 同步延迟（通常几秒到几分钟）
- Chrome 同步未启用扩展数据同步
- 网络问题

**解决方法：**
1. 检查 `chrome://settings/syncSetup` 确保已启用同步
2. 等待 5-10 分钟后再检查
3. 在 PC A 上点击扩展的同步按钮强制同步

### 问题 2: 配额超限
**症状：** 控制台显示 "QUOTA_BYTES_PER_ITEM quota exceeded"

**原因：** `chrome.storage.sync` 限制：
- 单项最大 8KB
- 总容量 102KB
- 最多 512 项

**解决方法：**
- 如果单词太多，需要分片存储或使用压缩

### 问题 3: 同步成功但 PC B 看不到
**症状：** PC A 显示同步成功，但 PC B 仍然为空

**可能原因：**
- PC B 的扩展版本旧（未重新加载）
- Chrome 同步服务器延迟
- 扩展 ID 不同（开发模式下每次加载 ID 会变）

**解决方法：**
1. 在两台 PC 上都重新加载扩展
2. 检查扩展 ID 是否相同（chrome://extensions/）
3. 考虑打包扩展以获得固定 ID
